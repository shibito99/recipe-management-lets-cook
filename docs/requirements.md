# 料理レシピ管理アプリ — 詳細仕様書

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| アプリ名 | 料理レシピ管理アプリ（仮称） |
| 対象ユーザー | 個人（料理を管理・記録したい方） |
| 目的 | 自分のレシピを一元管理し、材料・手順・栄養情報を素早く参照できる |
| 利用シーン | 自宅でのPC操作、料理中のスマートフォン操作 |

---

## 2. 機能要件

### F1 — レシピ登録

**概要**: 新しいレシピを登録する。

**入力項目**

| フィールド | 型 | 必須 | 制約 |
|-----------|-----|------|------|
| タイトル | 文字列 | ○ | 最大100文字 |
| 説明文 | テキスト | × | 最大500文字 |
| ジャンル | 選択肢 | ○ | 和食 / 洋食 / 中華 / イタリアン / その他 |
| タグ | 文字列（複数） | × | 1タグ最大20文字、最大10個 |
| 調理時間（分） | 数値 | × | 1〜999 |
| 人数（人前） | 数値 | ○ | 1〜99 |
| 材料 | 配列 | ○ | 最低1件。各行：食材名・分量・単位 |
| 手順 | 配列 | ○ | 最低1件。各行：テキスト（最大300文字）・任意画像 |
| メイン画像 | 画像ファイル | × | JPEG / PNG / WebP、最大5MB |

**正常系ユースケース**
1. ユーザーが登録フォームを開く
2. 各項目を入力する
3. 「登録する」ボタンを押す
4. バリデーション通過後、DBに保存
5. 登録完了メッセージを表示し、レシピ詳細画面へ遷移

**異常系・エラー処理**
- 必須項目未入力 → 該当フィールドにエラーメッセージを表示（フォーム送信はブロック）
- 画像ファイルサイズ超過 → 「5MB以下の画像を選択してください」と表示
- 画像フォーマット不正 → 「JPEG・PNG・WebP 形式のみ対応しています」と表示
- 通信エラー → 「保存に失敗しました。もう一度お試しください」と表示

---

### F2 — レシピ一覧表示

**概要**: 登録済みレシピをカード形式で一覧表示する。

**表示仕様**
- サムネイル画像（未設定時はデフォルト画像）
- タイトル
- ジャンル
- 調理時間
- 登録日

**ページング**
- 1ページあたり12件表示
- 無限スクロール or ページネーション（実装時に決定）

**並び替え**
- 新着順（デフォルト）
- タイトル昇順 / 降順
- 調理時間の短い順

**正常系ユースケース**
1. ユーザーがホーム画面を開く
2. 最新のレシピが一覧で表示される
3. カードをクリックするとレシピ詳細画面へ遷移

**異常系・エラー処理**
- レシピが0件 → 「まだレシピが登録されていません。最初のレシピを登録しましょう！」を表示
- データ取得失敗 → 「データの取得に失敗しました」と表示し、再試行ボタンを表示

---

### F3 — レシピ詳細表示

**概要**: レシピの全情報を表示する。

**表示内容**
- メイン画像
- タイトル・説明文・ジャンル・タグ・調理時間・人数
- 材料一覧（チェックボックス付き）
- 手順（番号付き、各ステップ画像があれば表示）
- 栄養情報（1人前）
- 編集・削除ボタン
- 「買い物リストに追加」ボタン

---

### F4 — レシピ編集・削除

**概要**: 登録済みレシピの内容を更新または削除する。

**編集**
- F1 と同じ入力フォームに既存データを初期値として表示
- 「更新する」ボタン押下で上書き保存
- バリデーションは F1 と同様

**削除**
- 削除ボタン押下で確認ダイアログを表示（「本当に削除しますか？この操作は取り消せません」）
- 確認後に削除実行、レシピ一覧画面へ遷移
- 関連する材料・手順・栄養情報も連動削除（CASCADE）

**異常系・エラー処理**
- 更新失敗 → 「更新に失敗しました。もう一度お試しください」と表示
- 削除失敗 → 「削除に失敗しました。もう一度お試しください」と表示

---

### F5 — レシピ検索・絞り込み

**概要**: キーワード・条件でレシピを絞り込む。

**検索方法**

| 種別 | 対象フィールド |
|------|--------------|
| キーワード検索 | タイトル・説明文・タグ（部分一致） |
| 食材名検索 | 材料の食材名（部分一致） |
| ジャンル絞り込み | ジャンル（完全一致） |
| タグ絞り込み | タグ（完全一致、複数選択可） |
| 調理時間絞り込み | ～15分 / ～30分 / ～60分 / 60分以上 |

**動作仕様**
- 検索バーへの入力は300msのデバウンスを設けてリアルタイム検索
- 複数条件はAND検索
- 検索結果0件の場合は「該当するレシピが見つかりませんでした」を表示

---

### F6 — 画像アップロード

**概要**: レシピ登録・編集時に料理写真をアップロードする。

**仕様**
- 対応形式: JPEG / PNG / WebP
- 最大ファイルサイズ: 5MB
- アップロード後にサーバーサイドでリサイズ（サムネイル用: 300×300px、詳細用: 800px幅）
- ドラッグ＆ドロップ または ファイル選択ダイアログに対応
- プレビューをアップロード前に表示

**異常系**
- サイズ超過・形式不正は F1 と同様のエラーメッセージ
- アップロード失敗 → 「画像のアップロードに失敗しました。再度お試しください」

---

### F7 — 栄養素計算

**概要**: 登録材料の分量から1人前の主要栄養素を計算・表示する。

**表示栄養素**

| 栄養素 | 単位 |
|--------|------|
| エネルギー | kcal |
| タンパク質 | g |
| 脂質 | g |
| 炭水化物 | g |
| 食物繊維 | g |
| 食塩相当量 | g |

**計算方法**
- 食品マスタテーブル（文部科学省「日本食品標準成分表」準拠）をDBに保持
- 材料名で食品マスタを検索し、分量・単位から100gあたりの成分値を按分して計算
- 材料名がマスタに存在しない場合は該当材料をスキップし、「一部の材料は栄養計算の対象外です」と注記

**表示タイミング**
- レシピ詳細画面に常時表示
- 登録・編集フォームでもリアルタイムプレビュー表示

---

### F8 — 買い物リスト連携

**概要**: レシピの材料を買い物リストに追加・管理する。

**機能詳細**

| 機能 | 説明 |
|------|------|
| 材料追加 | レシピ詳細画面から「買い物リストに追加」ボタンで一括追加 |
| 個別追加 | 材料ごとにチェックボックスで選択して追加することも可 |
| 人数変更 | 人数を変更すると分量を自動換算して追加 |
| チェック | 買い物リスト上でチェックボックスをONにすると購入済みとして表示 |
| 削除 | 個別削除 / 購入済み一括削除 |
| リスト管理 | 複数のリストを作成・切り替え可能（例：「今週の献立」「週末分」） |

**正常系ユースケース**
1. ユーザーがレシピ詳細画面で「買い物リストに追加」を押す
2. 人数入力ダイアログが表示される
3. 人数確認後、材料が買い物リストに追加される
4. 買い物リスト画面で追加された材料を確認・管理できる

## 3. 非機能要件

### 3-1. パフォーマンス

| 指標 | 目標値 |
|------|--------|
| レシピ一覧表示（1,000件） | ≤ 2秒 |
| 検索応答時間 | ≤ 1秒 |
| 画像アップロード（5MB） | ≤ 10秒 |
| 栄養素計算（材料20件） | ≤ 1秒 |

### 3-2. 可用性・信頼性

- 個人利用のため SLA は99%稼働を目標
- データはDBに永続化。ローカルキャッシュも活用してオフライン閲覧を検討

### 3-3. セキュリティ

- 個人利用アプリのため、初期段階は認証なし（ローカル環境想定）
- 将来的な公開を想定し、パスワード認証またはOAuth（Google等）を拡張ポイントとして設計
- 入力値はサーバーサイドでバリデーション（SQLインジェクション・XSS対策）
- 画像ファイルはMIMEタイプとマジックバイトを両方検証

### 3-4. UX / アクセシビリティ

- モバイルファースト設計（料理中のスマートフォン操作を想定）
- 最小タップターゲット: 44×44px（Apple HIG 準拠）
- フォントサイズ: 本文 16px 以上
- コントラスト比: WCAG AA 準拠（4.5:1 以上）
- ローディング中はスケルトンスクリーンまたはスピナーを表示

### 3-5. 保守性

- コンポーネント単位で責務を分割
- ビジネスロジックはUIから分離（サービス層 / ユースケース層）
- 環境変数で設定値を管理（DBの接続情報・画像ストレージの設定等）
