## 2. 選定理由

### フロントエンド: Next.js（静的エクスポート） + S3 + CloudFront

**採用理由**
- Next.js の静的サイト生成（SSG / CSR）を使い、`next export` でビルド成果物を S3 にアップロードする
- サーバーレスな構成のためEC2インスタンスが不要。運用コストと管理負荷を最小化できる
- CloudFront によるグローバルCDN配信でレスポンスが速い
- React ベースのため UI コンポーネント設計がしやすい

**制約と対処**
- SSR（サーバーサイドレンダリング）は使用不可 → データ取得はすべてクライアントサイドからRails APIを呼び出す
- Next.js の API Routes は使用不可 → バックエンドの処理はすべて Rails API に委譲する
- 動的ルーティングは `getStaticPaths` + `fallback: true` または CSR で対応する

### バックエンド: Ruby on Rails（API モード）

**採用理由**
- Rails の API モードは余分なミドルウェアを省いた軽量構成で、REST API に特化できる
- ActiveRecord による直感的なDB操作とマイグレーション管理が開発効率を高める
- RailsのActiveStorageを使いS3への画像アップロードをシンプルに実装できる
- 個人用途でのスループットには t3.micro で十分

**設計方針**
- JSON API のみを返す API モードで構築（ビュー不要）
- 認証: 初期フェーズは不要（個人利用・ローカル環境想定）。将来的に JWT（Devise + devise-jwt）で拡張
- CORS設定: Next.js の S3/CloudFront ドメインからのリクエストを許可

### データベース: MySQL 8.0 on RDS

**採用理由**
- RDS マネージドサービスにより、バックアップ・パッチ適用・フェイルオーバーを自動化できる
- MySQL 8.0 はウィンドウ関数・JSON型など現代的な機能を備えており、将来の機能拡張にも対応
- Rails の ActiveRecord と親和性が高い
- db.t3.micro（シングルAZ）で個人用途のコストを抑える

### インフラ管理: Terraform

**採用理由**
- AWSコンソールを直接操作せず、コードでインフラを定義・管理する（IaC）
- 構成変更の差分が明確になり、意図しない変更を防止できる
- 状態ファイル（terraform.tfstate）で現在のインフラ状態を追跡できる
- 将来のスケールアップや環境複製（本番/ステージング）が容易になる

## 3. AWS 構成概要

```
ユーザー（ブラウザ）
        │
        ▼
  CloudFront（CDN / HTTPS終端）
        │
        ├── /api/* ──────────▶ EC2 t3.micro
        │                         │
        │                    Nginx（リバースプロキシ）
        │                         │
        │                    Puma（Rails APIサーバー）
        │                         │
        │                    RDS MySQL db.t3.micro
        │
        └── /* ───────────▶ S3（Next.js 静的ファイル）

S3（画像ストレージ） ◀── Rails から直接 presigned URL で参照
```

**ネットワーク設計方針**
- VPC 内に EC2 と RDS を配置（RDS はプライベートサブネット）
- EC2 へのアクセスは CloudFront 経由のみに制限（セキュリティグループで制御）
- HTTPS は CloudFront で終端。EC2〜CloudFront 間は HTTP（VPC内通信）

## 4. 費用見積もり

| リソース | インスタンスタイプ | 月額（目安） |
|---------|----------------|-----------|
| EC2（Rails API） | t3.micro | ~$8 |
| RDS MySQL（シングルAZ） | db.t3.micro | ~$13 |
| S3（フロントエンド静的ファイル） | - | ~$0.5 |
| S3（画像ストレージ、1GB想定） | - | ~$0.5 |
| CloudFront（個人利用トラフィック） | - | ~$1 |
| データ転送・その他 | - | ~$2 |
| **合計** | | **~$25/月** |

> ※ AWS 無料利用枠（12ヶ月）適用時は EC2・RDS がそれぞれ750時間/月まで無料になる場合がある（t2.micro 相当）。

## 5. 将来拡張時の移行パス

| フェーズ | 変更内容 | 理由 |
|---------|---------|------|
| ユーザー公開 | 認証機能（JWT）追加 | 複数ユーザー対応 |
| アクセス増加 | EC2 を t3.small にスケールアップ | Rails API の応答性能確保 |
| 高可用性 | RDS をマルチAZ構成に変更 | 障害時の自動フェイルオーバー |
| スケールアウト | EC2 を Auto Scaling + ALB 構成に変更 | 水平スケーリング |
| Next.js SSR導入 | CloudFront + Lambda@Edge または ECS Fargate に移行 | SSR が必要な機能追加時 |
